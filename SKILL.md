# 微信群聊/私聊每日摘要生成器（通用版）

## 用途

当用户需要生成微信聊天记录摘要时使用。支持：
- 生成今天的微信摘要
- 补跑历史某天的摘要（自动化任务失败时手动补救）
- 补漏昨天的消息
- 列出所有群聊（供用户选择监控哪些群）
- 列出所有联系人（供用户选择监控哪些私聊）
- **首次使用时，AI 自动完成 wechat-cli 安装和配置，无需手动操作**

## 触发条件

用户说以下词语时触发：
- "微信摘要" / "微信日报"
- "补跑微信" / "补跑微信摘要"
- "生成微信摘要"
- "微信消息汇总"
- "微信补漏"
- "列出微信群" / "有哪些微信群"
- "列出微信联系人" / "有哪些微信联系人"
- "配置微信摘要" / "初始化微信摘要"

## 执行步骤

### 步骤 0：环境检测与初始化（首次使用）

**重要：以下步骤由 AI 自动完成，无需用户手动操作任何终端命令。**

#### 0.1 检测 Git 是否安装

```bash
git --version
```

- **如果成功（显示版本号）** → 继续 0.2
- **如果失败（找不到命令）** → 提示用户：

```
检测到 Git 未安装。Git 是安装 wechat-cli 的必要前提。

请先安装 Git：
- Windows: https://git-scm.com/downloads/win（下载安装包运行即可）
- macOS: brew install git
- Linux: apt install git / yum install git

安装完成后，重新触发此技能。
```

⚠️ Git 未安装时，**停止流程**，因为后续无法 clone wechat-cli 仓库。

#### 0.2 检测 PyYAML 是否安装

```bash
python -c "import yaml"
```

- **如果成功** → 继续 0.3
- **如果失败** → 自动执行以下命令安装：

```bash
pip install pyyaml
```

安装成功后继续 0.3。PyYAML 缺失不影响脚本运行（有 JSON 降级），但配置文件将使用 YAML 格式。

#### 0.3 检测 wechat-cli 是否已安装

```bash
wechat-cli --version
```

- **如果成功（显示版本号）** → 跳到 0.5
- **如果失败（找不到命令）** → 进入 0.4

#### 0.4 自动安装 wechat-cli（征求用户同意）

当检测到 wechat-cli 未安装时，向用户说明：

```
检测到 wechat-cli 未安装。wechat-cli 是读取微信聊天记录的必要工具。

安装内容包括：
1. 克隆 wechat-cli 源码到本项目的 wechat-cli/ 目录
2. 创建 Python 虚拟环境并安装依赖
3. 配置全局命令入口

整个过程约 2-3 分钟，不会影响你现有的任何环境。

我可以帮你自动安装吗？
```

**征求用户同意后**，执行以下命令：

```bash
cd <脚本所在目录> && python setup_wechat_cli.py
```

安装完成后，再次运行 `wechat-cli --version` 验证安装成功。

如果安装失败，告知用户错误信息并停止。

#### 0.5 检测 config.yaml 是否存在

检查脚本目录下是否存在 `config.yaml` 文件。

- **如果存在** → 跳到步骤 1
- **如果不存在** → 进入 0.6

#### 0.6 引导用户完成配置（AI 对话式）

当 `config.yaml` 不存在时，AI 通过**对话提问**收集用户配置偏好，然后**直接生成 `config.yaml` 文件**。用户无需手动编辑任何文件。

**问题 1：核心联系人**
```
首先，我需要知道你想监控哪些人的消息。

请输入核心联系人的名字，每行一个（可以写备注名或微信昵称），输入"完成"或空行结束。
例如：
  张三
  李四（产品组）

输入完成后，我会自动发现包含这些联系人的群聊。
```

**问题 2：群聊监控模式**
```
请选择群聊监控模式：
1. 自动发现（推荐）— 扫描所有群聊，自动识别并监控"与核心联系人共同存在的群聊"
   （解释：你指定了张三和李四，脚本会找出所有包含张三或李四的群，只监控这些群）
2. 监控所有群聊 — 监控你加入的全部群聊（消息量可能很大）
3. 指定群名 — 手动指定要监控的群聊（需要先运行"列出微信群"）

请输入 1、2 或 3：
```

- 如果选择 3，需要先运行 `python run_daily_v3.py --list-groups`，让用户挑选哪些群要监控

**问题 3：私聊监控模式**
```
请选择私聊监控模式：
1. 只监控核心联系人的私聊（推荐）
2. 监控所有私聊（消息量可能很大）
3. 指定联系人 — 手动指定（需要先运行"列出微信联系人"）

请输入 1、2 或 3：
```

**问题 4：摘要模式**
```
请选择摘要生成模式：
1. 规则模式（推荐）— 按关键词和时间结构汇总，速度快
2. AI 模式 — 使用 AI 模型生成摘要，需要安装 Ollama

请输入 1 或 2：
```

**收集完答案后，AI 直接生成 `config.yaml` 文件：**


生成后告知用户：`配置已保存到 config.yaml，可以随时手动编辑。`

### 步骤 1：解析用户请求

从用户的请求中提取目标日期和命令类型：

**日期解析**：
- "今天" / "今日" → 使用今天日期
- "昨天" / "昨日" / "补漏" → 计算昨天的日期
- "补跑X月X日的" → 解析为 YYYY-MM-DD 格式
- "补跑X月X日到X月X日的" / "X月X日到X月X日" / "最近N天" → 使用 `--date-range START END` 聚合模式
- 未指定日期 → 默认使用今天日期

**命令类型**：
- "列出微信群" / "有哪些微信群" → 执行 `--list-groups` 命令
- "列出微信联系人" / "有哪些微信联系人" → 执行 `--list-contacts` 命令
- "配置微信摘要" / "初始化微信摘要" → 跳到步骤 0.5，引导 AI 对话式配置
- "最近N天" / "这周" / "上周" → 计算日期范围后使用 `--date-range` 聚合模式
- 其他 → 执行摘要生成

**日期计算示例**：
- 使用 Python 动态计算当天和昨天日期，示例：若今天是某日，昨天就是前一天
- Python 代码：`(datetime.datetime.now() - datetime.timedelta(days=1)).strftime("%Y-%m-%d")`

### 步骤 2：检查运行环境

在执行操作之前，确保：

1. **微信 PC 版正在运行且已登录**
   - 如果未运行，提示用户：`微信 PC 版未运行，请先启动微信并登录，然后重新触发此技能。`

2. **config.yaml 已存在**
   - 如果不存在 → 回到步骤 0.6，引导 AI 对话式配置

3. **wechat-cli 可用**
   - 如果不可用 → 回到步骤 0.4，自动安装

### 步骤 3：执行命令

#### 3.1 列出所有群聊（`--list-groups`）

```bash
cd <脚本所在目录>
python run_daily_v3.py --list-groups
```

展示所有群聊列表，询问用户后续操作。

#### 3.2 列出所有联系人（`--list-contacts`）

```bash
cd <脚本所在目录>
python run_daily_v3.py --list-contacts
```

展示所有联系人列表，询问用户后续操作。

#### 3.3 生成单日摘要

```bash
cd <脚本所在目录>
python run_daily_v3.py --date YYYY-MM-DD
```

（将 `YYYY-MM-DD` 替换为步骤 1 计算的实际日期）

#### 3.4 生成多日聚合摘要（推荐：2-7天范围）

当用户请求"最近N天"或"X月X日到X月X日"时，使用 `--date-range` 聚合模式。此模式会生成**跨群聊的统一周期报告**，而非按天分文件。

```bash
cd <脚本所在目录>
python run_daily_v3.py --date-range YYYY-MM-DD YYYY-MM-DD
```

（两个日期均为包含，例如 `--date-range 2026-06-15 2026-06-21` 处理 6/15~6/21 共 7 天）

**输出文件**：
- `YYYY-MM-DD_to_YYYY-MM-DD_周期报告.md` — 主报告：全局概览、活跃排行、关键词、各聊天折叠摘要
- `YYYY-MM-DD_to_YYYY-MM-DD_<聊天名>.md` — 每个聊天的聚合摘要：日活跃趋势、时间线

**注意**：超过 7 天脚本会给出警告但仍可执行。

**临时指定群聊监控模式**（可选）：
```bash
python run_daily_v3.py --mode all --date YYYY-MM-DD
python run_daily_v3.py --mode discover --date YYYY-MM-DD
python run_daily_v3.py --mode specific --date YYYY-MM-DD
```

> **聚合模式 vs 单日模式**：单日按群生成独立 .md 文件；聚合模式生成一份统一周期报告+各聊天聚合摘要，适合补漏周末、休假回来、了解近期动态。

### 步骤 4：读取扫描结果

读取文件 `<输出目录>/YYYY-MM-DD_scan_result.json`

从中获取：
- 新发现的群聊列表（如果有）
- 每个群的摘要文件路径
- 每个私聊的摘要文件路径

### 步骤 5：读取各群聊/私聊摘要

根据 `scan_result.json` 中的路径，读取指定日期生成的各个 `.md` 摘要文件。

### 步骤 6：输出结构化摘要

将以下信息整合为结构化摘要：

#### 🆕 新发现群聊（如有）
列出动态扫描发现的新群聊（之前未监控的群），标注群名和包含的核心成员。

#### 微信群聊摘要
（来自各群的 `.md` 摘要文件）

#### 私聊摘要
（与核心联系人的私聊摘要）

## 重要注意事项

1. **AI 自动完成安装和配置**：首次使用时，AI 会自动检测环境、征求用户同意后安装 wechat-cli、通过对话引导用户完成配置。全程无需用户手动输入终端命令。

2. **config.yaml 由 AI 生成**：用户只需回答几个问题，AI 直接生成配置文件。

3. **run_daily_v3.py 支持 --date 参数**：格式为 `YYYY-MM-DD`

4. **微信 PC 版必须运行且已登录**：遇到此情况提示用户

5. **编码设置**：脚本内部已设置 UTF-8 输出（`PYTHONIOENCODING=utf-8`），Windows 下可额外设置 `PYTHONUTF8=0` 避免编码冲突

6. **多次日期处理**：逐个处理每个日期，最后汇总

7. **新发现群聊用 🆕 标记**

## 错误处理

| 错误情况 | 处理方式 |
|---------|---------|
| 微信 PC 版未运行 | 提示用户启动微信 PC 版并登录 |
| Git 未安装 | 提示用户安装 Git（https://git-scm.com/downloads），流程中止 |
| PyYAML 未安装 | AI 自动执行 `pip install pyyaml` |
| wechat-cli 未安装 | AI 征求用户同意后，自动运行 setup_wechat_cli.py |
| config.yaml 不存在 | AI 通过对话引导用户配置，然后生成 config.yaml |
| 脚本执行失败 | 显示错误信息，提供排查建议 |
| 未生成摘要文件 | 告知可能原因（微信未登录、无新消息等） |
| wechat-cli 安装失败 | 显示详细错误，建议用户检查网络和 Python 环境 |
